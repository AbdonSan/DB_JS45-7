El Modelo de Caja (Box Model)

En CSS, absolutamente todo es una caja. Cada etiqueta HTML (un parrafo, un titulo, una imagen, un boton) se renderiza como un rectangulo invisible al que podemos controlar su tamaño, su espacio interno, su borde y su separacion con otros elementos.

El Box Model (modelo de caja) es el concepto mas importante de CSS para entender el layout de una pagina. Se compone de 4 capas, de afuera hacia adentro:

1. Margin - Espacio externo transparente que separa la caja de otros elementos.
2. Border - El borde visible de la caja. Puede tener grosor, estilo y color.
3. Padding - Espacio interno entre el borde y el contenido. Es como el "acolchado".
4. Contenido - El texto, imagen o elementos que vive dentro de la caja.

En esta leccion vamos a explorar cada una de estas capas con ejemplos visuales e interactivos. Al final vas a poder controlar con precision como se ve y como se posiciona cualquier elemento.

Margin vs Padding

Margin = espacio FUERA de la caja (separa de otros elementos).
Padding = espacio DENTRO de la caja (separa el borde del contenido).

box-sizing: content-box vs border-box

Por defecto, width solo mide el contenido. El padding y border se suman al ancho total. Con box-sizing: border-box, el width incluye todo.

content-box (default)
width: 250px
+ padding: 20px
+ border: 5px
= 300px real
border-box
width: 250px
(padding y border incluidos)
= 250px real

Overflow (contenido desbordado)

Cuando el contenido es mas grande que la caja, overflow controla que pasa:

visible (default)

Este texto es muy largo y se desborda de la caja. El contenido se ve fuera de los limites del contenedor sin recortarse.

hidden

Este texto es muy largo y se desborda de la caja. El contenido que no cabe se oculta y no se puede ver.

scroll

Este texto es muy largo y se desborda de la caja. Aparece una barra de scroll para poder ver todo el contenido.

auto

Este texto es muy largo y se desborda de la caja. Solo muestra scroll si realmente se necesita.
